# 大陆农机配件网站开发需求规格说明书

> 适用对象：Codex / 前后端开发人员  
> 技术框架：Gin + GORM + MySQL + React  
> 目标：按照已确认的首页展示图效果，实现“大陆农机配件”网站前台展示与后台录入管理能力。

---

## 1. 项目概述

### 1.1 网站名称

**大陆农机配件**

### 1.2 网站定位

大陆农机配件网站是一个面向农机用户、维修门店、经销商、采购商和源头厂商的农机配件信息展示平台。

平台重点解决以下问题：

1. 在互联网上集中展示农机配件厂商及其产品信息。
2. 帮助用户通过分类导航快速查找农机配件。
3. 帮助用户浏览源头厂商、查看厂商主营产品、服务优势和官网入口。
4. 支持厂商配置自己的官网地址，用户点击后可跳转到厂商官网。
5. 通过后台管理系统维护导航菜单、厂商信息、配件分类、产品信息、首页推荐内容等。

### 1.3 首页实现目标

首页必须按照已确认的展示图效果实现，整体风格为：

- 蓝白配色；
- 现代、清爽、工业科技感；
- 左侧网站导航；
- 中间 Banner 搜索；
- 厂商卡片展示；
- 长页浏览更多厂商；
- 移动端优先适配；
- 不在首页展示“最新求购”模块。

## 2. 技术架构要求

### 2.1 后端技术栈

后端采用：

```text
Go + Gin + GORM + MySQL
```

要求：

1. Gin 作为 HTTP Web 框架。
2. GORM 作为 ORM 框架。
3. MySQL 作为业务数据库。
4. 后端提供 RESTful API。
5. 后端支持后台管理数据录入。
6. 前台展示数据全部从后端接口获取。
7. 初期可以保留 mock 数据，但最终必须具备 MySQL 数据录入与读取能力。

### 2.2 前端技术栈

前端采用：

```text
React + TypeScript + Vite
```

建议配套：

```text
React Router
Axios
CSS Modules / SCSS / 普通 CSS 均可
```

要求：

1. 前端页面必须响应式适配 PC 和移动端。
2. PC 端按照已确认首页效果图实现。
3. 移动端按照右侧手机效果实现。
4. 前端所有展示数据通过接口获取。
5. 首页结构、菜单、厂商卡片、产品信息均应由后端配置。

从“首页”到“待扩展菜单”为主导航区域。后期如果继续增加一级菜单，该区域必须支持上下滚动。

也就是说：

```text

  变速箱齿轮

液压系统配件
动力发动机配件
制动换挡配件
待扩展菜单
```

以上内容全部属于左侧主导航区域。

### 6.3 左侧导航菜单结构

一级菜单：


### 6.4 左侧导航交互要求

1. 一级菜单支持点击。
2. 有二级菜单的一级菜单支持展开和收起。

4. 一级菜单右侧显示展开 / 收起箭头。
5. 二级菜单缩进显示。
6. 当前选中菜单高亮蓝色。
7. 左侧主导航区域支持滚动条。
8. 滚动条要细，不影响美观。
9. “提交厂商、友情链接、关于平台”不属于主导航，放在主导航下方的辅助链接区域。

## 14. 数据库设计建议

### 14.1 menu 导航菜单表

```sql
CREATE TABLE menus (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  parent_id BIGINT DEFAULT 0,
  icon VARCHAR(100),
  menu_type VARCHAR(50) NOT NULL,
  path VARCHAR(255),
  sort_order INT DEFAULT 0,
  is_enabled TINYINT DEFAULT 1,
  is_top TINYINT DEFAULT 0,
  is_default_open TINYINT DEFAULT 0,
  created_at DATETIME,
  updated_at DATETIME,
  deleted_at DATETIME NULL
);
```

### 14.2 vendors 厂商表

```sql
CREATE TABLE vendors (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(150) NOT NULL,
  short_name VARCHAR(100),
  logo VARCHAR(255),
  cover_image VARCHAR(255),
  province VARCHAR(50),
  city VARCHAR(50),
  county VARCHAR(50),
  address VARCHAR(255),
  main_products VARCHAR(500),
  service_models VARCHAR(500),
  service_advantages VARCHAR(500),
  description TEXT,
  website_url VARCHAR(255),
  phone VARCHAR(50),
  wechat VARCHAR(100),
  contact_name VARCHAR(50),
  is_recommended TINYINT DEFAULT 0,
```

## 19. 首页禁止项

首页主体区域禁止出现：

```text
最新求购
求购列表
求购信息流
采购需求动态
```

## 20. 验收标准

### 20.1 PC 端验收

1. 首页整体效果接近已确认展示图。
2. 顶部导航、左侧导航、Banner、推荐厂商、更多厂商、入驻引导、底部统计完整展示。
3. 左侧导航从“首页”到“待扩展菜单”为一个主导航区域。

5. “传动配件”默认展开，并展示 7 个二级菜单。
6. “提交厂商、友情链接、关于平台”在左侧辅助链接区域展示。
7. 首页不出现“最新求购”模块。
8. 推荐厂商一行展示 5 个卡片。
9. 更多厂商至少展示 2 行卡片。
10. 厂商卡片中的“进入厂商”按钮可以跳转。

1. 移动端展示效果接近确认图右侧手机效果。

3. 有搜索框。
4. 分类快捷入口清晰展示。
5. 推荐厂商和更多厂商为竖向信息流。
6. 底部导航固定。
7. 页面内容不被底部导航遮挡。

### 20.3 后台验收

1. 可录入和维护导航菜单。
2. 可录入和维护厂商信息。
3. 可配置厂商官网地址。
4. 可设置推荐厂商。
5. 可维护产品和分类。
6. 前台展示内容从后端接口读取。
7. 新增一级菜单后，前台左侧导航可正常展示并支持滚动。
